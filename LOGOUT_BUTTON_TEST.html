🧪 Test de Botón Logout

✅ Qué Está Pasando

He mejorado el sistema de logout con logging exhaustivo. Ahora cada paso está documentado y visible en la consola del navegador.

🔍 Cómo Verificar que el Botón Funciona

  1. Login: Accede a https://connect-ar-hr-main.vercel.app/ con owner@conectar.com
  2. Abre DevTools: Presiona F12 y ve a la pestaña Console
  3. Busca el botón:
    • Sidebar: Esquina inferior izquierda (ícono LogOut)
    • Dropdown: Esquina superior derecha > click en avatar > "Cerrar Sesión"
  4. Click en el botón y observa los logs

📊 Logs que Deberías Ver

[AppShell] Iniciando logout... [AppShell] Limpiando sessionStorage... [AppShell] SessionStorage limpiado [AppShell] Llamando API logout... [AppShell] Logout API retornó status: 200 [AppShell] Logout API exitosa: { success: true... } [AppShell] Esperando 100ms antes de redirigir... [AppShell] Redirigiendo a /login...

🔗 Verificaciones en Network Tab

  1. Abre DevTools → Network tab
  2. Haz click en Logout
  3. Busca la solicitud POST a /api/auth/logout
  4. Verifica:
    • Status: 200 ✅
    • Response: {"success": true, "redirect": "/login"}
    • Headers: Cookie conectar_session se debe eliminar

🚨 Si Algo Falla

Síntoma Probable Causa
Sin log "[AppShell] Iniciando logout..." Botón no clickeado o handler no registrado
Logs se detienen en "Llamando API logout..." Servidor no responde (timeout)
Status 500 en logout API Error al decodificar sesión
Error "Error en logout:..." Excepción en el cliente (revisa stack trace)

⚠️ Nota sobre Cookies

Si ves un error sobre cookies en navegadores de incógnito/privado, es normal. Para testing completo, usa una ventana normal del navegador.

📋 Mejoras Implementadas

  • Client-side (app-shell.tsx):
    • Logging detallado de cada paso del logout
    • Timeout de 5 segundos para API call
    • Error handling completo con stack traces
    • Response parsing con fallback
  • Server-side (logout/route.ts):
    • Logging de recepción de solicitud
    • Verificación de sesión y decodificación Base64
    • Logging de eliminación de cookie
    • Error details en response
